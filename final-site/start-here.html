Start Here

If you are just starting out in the process of learning how to build a website or webpage from scratch, it is highly important to know how to properly use HTML and CSS, the fundamental building blocks of webpages. The next two sections below give a overview of HTML, and an overview of CSS respectively.

HTML

In website design, HTML can be considered the most important fundamental building block of a webpage, as HTML acts as the structural "markup" layer of a webpage. HTML is used for holding and structuring the actual raw information of a webpage, as well as defining certain global traits of a page to communicate aspects of the page to browsers and web crawlers at a glance. In the markup of a webpage, everything is encapsulated in an "<html>" element, followed by the two major sections of the markup, the "<head>", used for storing global traits of a page, and the "<body>", used for storing everything else, namely the content to be shown/used in the page.

CSS

In website design, CSS is a highly important building block of a webpage, as CSS acts as the presentational "style" layer of a webpage. CSS is used as a way to address elements for uniform or unique styling choices in a webpage, and can even allow for certain forms of interactability, without the use of heavier-handed solutions such as using JavaScript. Due to the fact that CSS is typically "linked" from the HTML head of the page it will be used to style, CSS is also highly reusable, as various pages of a website can all link back to a single, unified CSS file for styling. CSS additionally offers methods of responsive presentation, by allowing for elements of the page to be styled based on the width and height of the current viewport.

Progressive Enhancement

When designing a website, an important concept to keep in mind before starting to build a webpage is the idea of progressive enhancement. Progressive enhancement is the idea that a website should be constructed in an order such that fundamental, structural aspects of a site should precede, and be independent of, enhancement features such as presentational or functional aspects. The reasoning behind this methodology is that a website should exist primarly for the content it hosts, and so that content shouldn't be as subject to change based on circumstances as things like the presentation of the site, which could easily change with aspects such as viewport dimensions, or even just a preference of light or dark display modes. This idea of progressive enhancement is also greatly important for compatability and performance, since not every device or browser supports the same presentational or functional features, but the structure of information on a website can generally be relied on to be preserved. Progressive enhancement generally dictates that the progression of site layers will start with the structural layer, HTML, followed by the presentational layer, CSS, followed then by the functional layer, which is generally JavaScript.
