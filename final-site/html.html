HTML

Introduction

In website design, HTML can be considered the most important fundamental building block of a webpage, as HTML acts as the structural "markup" layer of a webpage. HTML is used for holding and structuring the actual raw information of a webpage, as well as defining certain global traits of a page to communicate aspects of the page to browsers and web crawlers at a glance. In the markup of a webpage, everything is encapsulated in an <html> element, followed by the two major sections of the markup, the <head>, used for storing global traits of a page, and the <body>, used for storing everything else, namely the content to be shown/used in the page. The <html> element additionally has an attribute for defining the language of a page, such as lang="en" for English.

Head Elements & Attributes

Title

The title of a page can be specified with the <title> element. For instance, the current page uses the following for its title:
<title>HTML & CSS Site Project - HTML</title>

Character Set

To define a character set for a given page, use the <meta> metatag with a charset attribute. There are various character sets to choose from, but a common pick is "utf-8", leading to the head element:
<meta charset="utf-8">

Description

The description of a page can be defined using the <meta> metatag with a name="description" attribute, and a content attribute to define the description. This is typically used by browsers to show a quick description of what the page is about. For an example, the current page uses the following for its description:
<meta name="description" content="Matthew Ingram's Mobile Web Development Final Project HTML page"/>

Author

The author of a page can be defined using the <meta> metatag with a name="author" attribute, and a content attribute to define the author. For an example, the current page uses the following for its author:
<meta name="author" content="Matthew Ingram"/>

Viewport

The viewport of a page can be defined using the <meta> metatag with a name="viewport" attribute, and a content attribute to define its behavior. The viewport metatag is primarily important for responsive design with respect to differently sized devices. For an example, the current page uses the following standard format for the metatag:
<meta name="viewport" content="width=device-width, initial-scale=1"/>

Linking

The page can be linked to other resources, such as a stylesheet or shortcut icon, using the <link> element, which is paired with a rel attribute defining the link's relationship with page contents, and a href attribute defining the location the link goes to. For instance, one way to link a stylesheet located in a "css-files" directory would be as follows:
<link rel="stylesheet" href="css-files/style.css"/>
Another example would be for how to link up a shortcut icon, such as one located in an "img-files" directory:
<link rel="shortcut icon" href="img-files/favicon.ico"/>

Body Elements

Header

The <header> element relates to introductory or navigational content, a major heading.

Main

The <main> element encapsulates the primary content of a page, typically being preceeded by a <header> element, and being followed by a <footer> element.

Sections

The <section> element encapsulates a group of other elements, all of which follow the same theme, such as paragraphs and figures about a topic.

Articles

The <article> element encapsulates a group of other elements, all of which follow the same theme, and are self-contained within the scope of the <article>, such as the text and images of a single news article.

Asides

The <aside> element encapsulates a group of other elements which follow the same general theme as surrounding elements, but does not have to directly tie in to the other elements. An <aside> element should not be greatly important to the main flow of a page, it should be completely fine to remove the element without majorly disrupting the experience of the page.

Footer

The <footer> element relates to additional information that closes out another element, such as links to related content or copywrite information.

Navs

The <nav> element relates to navigational elements, typically a list of links to parts of the current page, or to other pages.

Body Attributes

Element Class

An element can be given a class attribute for selection via CSS in order to style only elements of that class. For example, adding a "color-this-red" class to a <p> element can be done like so:
<p class="color-this-red">Text goes here!</p>

Element ID

An element can be given an id attribute for selection via CSS in order to style only one unique element, which differs from class attributes which may be used on multiple elements. For example, adding a "page-title" id to a <h1> element can be done like so:
<h1 id="page-title">Insert page title here!</h1>

Element Hyperlink

An element can be given an href attribute to indicate a hyperlink location for the element to point to, usually via creating a link with the <a> element. The href attribute can also be used to link to elements with an id attribute, or back to the top of the current page. An example showing how to link back to the top of the current page is as follows:
<a href="#top">Back to Top</a>

Element Title

An element can be given a title attribute to allow for a text popup to show up after hovering over the element. An example of a title attribute in code is as follows:
<p title="This is a title!">This text has a title!</p>
The element generated by the previous code is as follows:

This text has a title!

Image Source

An <img> element can be given a src attribute to indicate the location of the image to be displayed. An example of linking up an image stored in an "img" directory titled "cool-image" would be as follows:
<img src="img/cool-image"/>

Image Alternate Text

An <img> element can be given an alt attribute to provide alternate text for an image, which can help to provide text for screen readers. An example of alt text being added to the previous image in the "Image Source" section could be:
<img src="img/cool-image" alt="A very cool image"/>

Responsive Images

Image Source Set

An <img> element can be given a srcset attribute to indicate the locations of the different images to be displayed, where the different images are usually different sizes of the same image. An example of linking up the images for a srcset where all images are in an "img" directory, and are titled as "responsive" along with their size in pixels is as follows:
<img srcset="img/responsive-1000px.jpg 1000w, img/responsive-800px.jpg 800w, img/responsive-500px.jpg 500w"/>
Each image image source is paired with its size in "w" width units, and the list of the source-width pairs is comma-separated.

Image Sizes

An <img> element can be given a sizes attribute along with a srcset attribute to indicate the intended display sizes of the image, this is primarily useful for having the image be responsive to CSS breakpoints. An example of a sizes attribute where the image element has a breakpoint of "64rem" at which it will have a width of "50vw", and otherwise the image will be "100vw", is as follows:
<img srcset="img/responsive-1000px.jpg 1000w, img/responsive-800px.jpg 800w, img/responsive-500px.jpg 500w" sizes="(min-width: 64rem) 50vw, 100vw"/>

Microdata

Item Scope

The itemscope attribute indicates that an item is created and defined by this element and its descendents.

Item Type

The itemtype attribute indicates what type of item is to be described and its properties context by linking a vocabulary by URL.

Item ID

The itemid attribute indicates a unique identifier for the item.

Item Property

The itemprop attribute indicates the property that an element will satisfy of the item described by the vocabulary.

Item Reference

The itemref attribute indicates non-descendent elements that satisfy properties of the item described by the vocabulary, this is given by a list of ids to identify those other elements.
